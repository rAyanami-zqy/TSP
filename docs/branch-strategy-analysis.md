# 分支策略分析

## 穷举策略的缺陷

### 1. 搜索空间重叠（根本性缺陷）

穷举策略在每个节点对**所有**未决边创建子节点，但子节点之间**不是划分关系**——它们的搜索空间相互重叠。

举例：n=5，有 3 条未决边 {e1, e2, e3}。最终状态 {force e1, force e2} 可以通过两条不同路径到达：

```
Root → force e1 → force e2
Root → force e2 → force e1
```

同一个状态被重复探索。对于有 K 条未决边的节点，一个 k 条 force 边的赋值可以通过 k! 条不同路径到达。这导致搜索树远大于 2^K——接近 O(K! · 2^K)，大量冗余。

从实测数据可直观看出：n=4 时，穷举策略创建 1761 个节点，Smart 策略仅需 6 个。差距约 300 倍。

### 2. 剪枝效率低

Smart 策略每次选择"最可能违反约束"的边进行分支，force 该边会立即提高 1-tree 下界，收紧 bound。而穷举策略分散在所有边上，每条决策路径深度增长慢，下界提升缓慢，剪枝触发延迟。

### 3. 内存爆炸

每个节点创建 2×K 个子节点（存放在 vector 中），DFS 也无法避免大量 pending children 驻留内存。n=5 时已无法在合理时间内完成。

## 完备性分析

### Smart 策略：完备

```
对任意节点 N，选未决边 e：
  - 左子：force e  → 包含 e 的所有 tour
  - 右子：forbid e → 不含 e 的所有 tour
```

这是标准的**二元划分**（binary partition）：

- 两个子空间**互斥**（disjoint）：一条 tour 不可能同时包含和不包含 e
- 两个子空间**穷尽**（exhaustive）：任意 tour 对 e 只有这两种可能
- 递归到叶子时，所有边均被决定（forced 边 = tour 边集，forbidden = 其余）

**1-tree 下界保证**：任意 TSP tour 去掉一条与顶点 0 关联的边后即为一棵生成树 + 两条与 0 的边，所以最小 1-tree 的代价 ≤ 最小 tour 的代价。下界有效 → 剪枝安全（不会剪掉最优解）。

**结论：Smart 策略是完备的，且搜索空间无冗余。**

### 穷举策略：完备，但冗余

```
对任意节点 N，有未决边集 {e1, ..., eK}：
  对每个 ei，创建 force_ei 和 forbid_ei 共 2K 个子节点
```

完备性成立的理由：

- 任意 tour 对 ei 要么 force 要么 forbid
- 对所有 ei 的每种组合，都存在至少一条路径到达该赋值
- 当所有边都被决定后，`branch_candidates` 为空，搜索自然终止
- 最优解一定在某个叶子节点被访问到

但子节点之间**不互斥**（force_e1 和 force_e2 的搜索空间有交集——都存在 `{force e1, force e2}` 的赋值），因此同一状态可通过不同排序被多次探索。这不是"搜索更彻底"，而是"同一件事重复做了很多次"。

## 对比总结

| | Smart | Exhaustive |
|---|---|---|
| 完备性 | 完备 | 完备 |
| 子节点互斥 | 互斥（划分） | 不互斥（覆盖但重叠） |
| 搜索空间 | 2^|E| 个叶子 | 远大于 2^|E|（重复探索） |
| 分支因子 | 2 | 2K |
| 边选择策略 | 启发式（最大度数顶点） | 无（全部枚举） |
| 实用性 | 实用（n≤12） | 仅 n≤4 可行 |
